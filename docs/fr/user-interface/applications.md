# Menu des Applications

<br>

## Composants

Cette section contient une liste complète de tous les composants disponibles dans le système. À partir d'ici, vous pouvez consulter les détails de chaque composant, ainsi qu'ajouter de nouveaux composants ou multi-composants, élargissant ainsi les capacités de votre application. Idéal pour organiser et gérer tous les composants de votre projet.

![Composants](../assets/images/user-interface/components.png)
<br>

## Modules Python {: #python-modules }

Cette section offre la possibilité d'ajouter et de gérer des modules Python courants. Ces modules deviennent disponibles à la fois sur le front-end dans le Script de Composant et sur le back-end dans le Script d'Exécution pour le flux de données. Cela vous permet de gérer de manière centralisée la logique qui peut être utilisée dans différentes parties de l'application. Lisez-en plus ici : [Utiliser Python](../app-development/using-python.md)

![Modules Python](../assets/images/user-interface/python-mod.png)
<br>

## Intégrations

La section "Intégrations" joue un rôle clé dans la création de liens entre votre application et des services externes. Ici, vous pouvez ajouter de nouvelles intégrations, supprimer celles qui ne sont pas nécessaires et modifier les paramètres des intégrations actuelles, ce qui offre flexibilité et évolutivité à la fonctionnalité de l'application.

![Menu des Intégrations](../assets/images/user-interface/integrations-menu.png)
<br>

## Publication

Une fois que vous avez terminé de configurer et de préparer vos composants, cette section vous permet de les publier pour une utilisation future. Vous pouvez choisir des composants, des localisations, des intégrations et même des modules Python à inclure dans votre publication, rendant le processus de déploiement efficace et organisé. Lisez-en plus ici : [Publier des Composants](#publication)

![Publication](../assets/images/user-interface/publication.png)
<br>

## Exportation/Importation

Cette section fournit des outils pour exporter et importer des configurations de composants au format JSON. Cela est utile pour déplacer des paramètres entre différents environnements, tels que du développement au test ou à la production, facilitant ainsi la migration et la synchronisation des modifications entre différentes instances de Studio.

![Exportation/Importation](../assets/images/user-interface/export-import-new.png)
