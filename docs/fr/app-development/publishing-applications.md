# Publications

Pour créer une version de composant, une localisation, une extension ou toute autre personnalisation appropriée via la plateforme, vous devez la publier. Et pour qu'une version de composant soit publiée, elle doit être marquée comme prête à être publiée.

Le processus de publication est contrôlé par la machine d'état, qui vous permet de contrôler efficacement toutes les étapes et, en cas d'erreurs, de ramener le système à l'état précédent. Une fonctionnalité de verrouillage global a été introduite pour empêcher le démarrage de plusieurs publications en même temps. Les utilisateurs voient le statut des publications actives, et après chaque publication terminée, les statuts des objets prêts à être publiés sont mis à jour automatiquement.

**Étapes pour publier un composant :**

- **Bouton “Enregistrer”** : Utilisé pour enregistrer les modifications actuelles du composant.
- **“Prêt à Publier”** : Marque un composant comme prêt à être publié après que toutes les modifications ont été enregistrées.

**Étapes pour publier des localisations et des intégrations :**
- Devenir automatiquement disponible pour publication après que vous avez effectué des modifications.

**Processus de Publication :**
1. **Aller à Publications** : Situé dans ‘Studio→Applications→Publication.’
2. **Sélection des éléments à publier** : 
   - Les composants, localisations, intégrations et modules Python sont sélectionnés pour être publiés.
3. **Publication finale** : 
   - La procédure est complétée en cliquant sur le bouton de publication.
   - Une notification apparaît lorsque la publication est réussie.
